# Track Decision

## My Track
I have chosen the **DevOps** track for the remainder of the bootcamp.

## Three Signals That Pointed Me Here
1. **The Infrastructure Environment Hooked Me:** Successfully spinning up my native Ubuntu WSL subsystem environment and managing system updates via package tools like `apt` felt incredibly rewarding. Interacting directly with the underlying Linux kernel layer made me realize how much I enjoy system operations over writing application layouts.
2. **System Control and Process Operations Feel Natural:** Working through the hands-on process management lab exercises—such as tracing active application threads using commands like `ps aux` and safely terminating hung processes with system signals—felt intuitive and interesting. I highly enjoy the puzzle of monitoring server diagnostics and optimizing resources.
3. **Strong Industry Market Alignment:** Looking closely at the modern tech ecosystem, there is an essential and expanding demand for infrastructure engineers who understand server isolation, security bitmasks, and stable development pipelines. Specializing in DevOps builds highly tradeable skills that directly match cloud administrator roles.

## The Runner-Up I Considered
I seriously considered the **Backend** development track because of my foundation working with data array logic, conditional execution structures, and core programming functions. However, I ruled it out because I found managing permissions, isolating server workflows, and maintaining systems infrastructure to be more engaging than drafting raw application data logic full-time.

## What I'm Nervous About
I am specificially nervous about wrapping my head around automated CI/CD pipeline structures and complex networking concepts, particularly subnet configurations and gateway routing rules. I plan to dedicate extra study hours to foundational cloud networking concepts during our initial system automation weeks.

## Commitment to Myself
By the end of Phase 2 (Month 6), I commit to building, testing, and fully documenting three distinct infrastructure automation projects on my GitHub, each featuring secured Linux user groups, structured permissions configurations, and automated deployment actions.
